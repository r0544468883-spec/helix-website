# Helix Project Starter

זה ה-package הראשוני לבניית האתר של Helix עם Claude Code. כל הקבצים, הברנד, הקופי, וההחלטות שלקחנו במהלך התכנון מאורגנים כאן.

## איך מתחילים 

### שלב 1: התקנות מקדימות

```bash
# וודא ש-Node.js מותקן (גרסה 18+)
node --version

# וודא ש-npm או pnpm מותקן 23
npm --version
```

### שלב 2: שמור את התיקייה הזו במקום קבוע

```bash
# למשל:
~/Projects/helix/
```

### שלב 3: התחל סשן Claude Code בתיקייה

```bash
cd ~/Projects/helix
claude
```

Claude Code יקרא אוטומטית את `CLAUDE.md` ויבין את כל הקונטקסט של הפרויקט.

### שלב 4: התחל לבנות

הפרומפט הראשון המומלץ ל-Claude Code:

```
תקרא את כל הקבצים בתיקייה, התחיל ב-CLAUDE.md.
אחרי שיש לך תמונה מלאה, אתחל פרויקט Astro חדש בתיקייה הזו
לפי המפרט ב-CLAUDE.md, והתחל לבנות את דף הבית.
```

## דברים שחשוב לדעת לפני שמתחילים

1. **לא להציג מחירים באתר.** הוחלט אסטרטגית - מחירים נקבעים בפגישת אפיון יחד עם הלקוח
2. **לא לעצב מחדש.** ה-mockups עברו 6 איטרציות. תפקיד Claude Code הוא לפורט אותם נאמנה ל-Astro
3. **Hebrew RTL.** השפה הראשית עברית. כל קומפוננטה צריכה לתמוך RTL מההתחלה
4. **Heebo font.** הגופן היחיד באתר. אל תוסיף עוד גופנים
5. **#10B981 emerald.** הצבע היחיד שנעשה בו שימוש דקורטיבי, ורק על אלמנטים ניתנים-ללחיצה

## בעיות נפוצות שצריך להימנע מהן

הברנד הזה נבנה בקפידה להיות **שונה מהאתר הסטנדרטי שמרגיש AI-מיוצר**. ה-mockups כבר פותרים את הבעיות האלה, אבל הנה רשימת בעיות שצצו במהלך התכנון, כדי שאם Claude Code יתחיל לסטות - תזהה את זה:

- ❌ סקציה כהה דרמטית ב-CTA (ניסינו, החלטנו שלא)
- ❌ כפתורים עם זוהר ירוק חזק
- ❌ pulsing dots ב-hero
- ❌ "מסלולים" במקום "שירותים"
- ❌ "מאחורי הברנד" במקום "מי אני"
- ❌ "Let's Talk —" עם קווי עיטור
- ❌ Em-dashes בכל מקום (מותר אחד לכל סקציה לכל היותר)
- ❌ Em-dash בעברית (` — ` נראה תמיד גרוע בעברית)
- ❌ "01 / METHOD" decorations
- ❌ מבנה חוזר של [תג] [כותרת] [תיאור] [גריד] בכל סקציה

## תהליך הפיתוח המומלץ

1. **התחל קטן.** קודם דף הבית בלבד, ואז נוסיף.
2. **שתף בכל שלב.** אחרי שבונים סקציה, רענן את התצוגה ובדוק שזה תואם ל-mockup.
3. **בדוק על מובייל.** הלקוחות שלך גולשים מהטלפון. כל החלטה עיצובית צריכה לעבוד שם.
4. **דחה optimization.** קודם נכון ויפה, אחר כך מהיר.

## 3. אפיון פונקציונלי — מה המערכת עושה

### 3.1 Onboarding (15 דקות, בלי קוד)
- **חיבור מקורות דאטה** (§3.1.1) — Google Search Console + ספק keyword-data (Semrush/Ahrefs).
- זיהוי פלטפורמת האתר (CMS auto-detect).
- **בחירת שפת תוכן** — עברית (ברירת מחדל) / אנגלית / שתיהן. auto-detect משפת האתר, ניתן לשינוי. קובע שפת מחקר, כתיבה, ו-prompt panel.

#### 3.1.1 מקורות דאטה (Connectors)
| מקור | מה נותן | הערה |
|---|---|---|
| **Google Search Console** (read-only) | הדאטה **שלך** — impressions, clicks, position, ביטויים שכבר מדורגים | חובה. בסיס למעקב וגילוי הזדמנויות "כמעט-שם" |
| **Semrush API** | volume, keyword difficulty, ביטויי מתחרים, backlinks, SERP features | ספק ה-keyword-data הראשי. חיבור מפתח API של הלקוח, או שכבה משותפת של HELIX |
| **Ahrefs API** | חלופה/השלמה ל-Semrush (difficulty, מתחרים, backlinks) | אופציונלי — לפי מה שיש ללקוח |
| **חלופה זולה** (DataForSEO / Serpapi) | volume/SERP במחיר נמוך per-call | אם לא רוצים מנוי Semrush/Ahrefs מלא |
| **Google Analytics 4** (אופ') | תנועה, המרות | להעשרת דוחות/ROI |
- **GSC לבדו לא מספיק** — הוא לא נותן volume/difficulty לביטויים שעוד לא מדורגים עליהם. Semrush/Ahrefs הם מקור ה-research האמיתי.
- **סריקת אתר + מתחרים** (FireCrawl — כבר מוטמע ב-`helix/lib/geo-scan.ts`): מיפוי תוכן קיים, פערים, ומתחרים בנישה.
- **אשף אסטרטגיה:** ה-AI בונה אסטרטגיית תוכן רבעונית עם מילות מפתח יעד — המשתמש מאשר.

### 3.2 מנוע מחקר (Research Engine)
- **מקור דאטה:** GSC (הדאטה שלך) + **Semrush/Ahrefs API** (volume, difficulty, מתחרים) — §3.1.1.
- **מחקר מילות מפתח** לפי כוונת-קנייה (buyer-intent), לא רק volume. **Keyword Opportunity Score** = `(Volume × Intent × ClickRate) / Difficulty` — מזין מ-Semrush.
- **ניתוח מתחרים** — keyword gap (ביטויים שהם מדרגים עליהם ואנחנו לא) דרך Semrush/Ahrefs; חולשות ופערי תוכן.
- **גילוי ביטויים שבועי** אוטומטי — המערכת מזהה הזדמנויות חדשות (GSC "כמעט-שם" + הזדמנויות Semrush) בלי בקשה.
- **פילוח כוונה** (informational / commercial / transactional) לתעדוף.

### 3.3 מנוע תוכן (Content Engine) — הליבה
- **כתיבת מאמרים 2,000+ מילים** מותאמי-SEO, **דו-לשוני:** בעברית אנושית (Hebrew Content by Helix + Claude + הגהת **baldiga חובה**), או באנגלית (Claude, פרוזה סטנדרטית — baldiga לא רלוונטי). שפה per-אתר לפי §3.1.
- **תמונות AI** לכל מאמר (יצירה אוטומטית + alt-text).
- **מבנה סכמה** (Schema.org / JSON-LD) אוטומטי — Article/FAQ/HowTo לפי סוג התוכן.
- **תזמון פרסום** — 4-14 מאמרים בחודש לפי החבילה, drip אוטומטי.
- **Human-in-the-loop אופציונלי:** אישור לפני פרסום, או autopilot מלא — בחירת המשתמש.

### 3.4 מנוע פרסום רב-CMS (Publishing Engine) — הבידול המרכזי 🔌
פרסום אוטומטי לכל פלטפורמה דרך **adapter per-CMS** (ליבה אחת, מתאמים דקים):

| פלטפורמה | שיטת פרסום |
|---|---|
| **WordPress / Elementor** | WordPress REST API (posts, media, categories, tags, Yoast/RankMath meta) |
| **Wix** | Wix Blog / Data API |
| **Webflow** | Webflow CMS API (collections) |
| **Next.js / Headless** | Webhook + Git commit (MDX) או Supabase content API → אתר מושך |
| **קוד פתוח / מותאם** | Generic webhook + REST endpoint (מתועד ללקוח) |
| **אתרי הסוכנות** | אינטגרציה ישירה עם ה-CMS ש-HELIX/הקולגה בונים |

- כל adapter מפרסם: כותרת, גוף, תמונות, meta-tags, schema, קטגוריות, `llms.txt`.
- **auto-detect** בזמן onboarding בוחר את ה-adapter הנכון; ידני כ-fallback.

### 3.5 מנוע GEO/AEO (אופטימיזציה למנועי AI) — בליבה 🤖
- כל מאמר **נכתב מראש לציטוט** ב-ChatGPT/Perplexity/Claude/Gemini (מבנה שאלה-תשובה, ציטוטים, ישירות).
- **`llms.txt` + `llms-full.txt` אוטומטיים** — קבצים שמאפשרים למודלי AI לגשת לתוכן (מלא ומקוצר).
- **מבנה ל-Google AI Overviews** — schema + תשובות ישירות שמזכות ב-featured snippet / AI Overview.
- **"ציטוטים ברמת ויקיפדיה":** הסוכן **מהנדס לתוך כל עמוד מקורות סמכותיים** (סטטיסטיקות, מחקרים, ציטוטים מיוחסים) — כדי שה-LLM יעדיף לצטט *אותנו* כמקור. תוכן quotable, לא רק readable.

#### 3.5.1 מנוע Citation-Gap → Patch (הליבה האגנטית) 🎯
לא רק מדידה — **תיקון פרואקטיבי**. הסוכן רץ בלולאה אוטונומית (בהשראת enso/Hermes, אך עברית-first):
1. **סורק** את מנועי ה-AI על **שאילתות-קונה אמיתיות** בנישה (prompt panel בעברית).
2. **מזהה פערי ציטוט** — "משבצות לא-מאוישות": שאלות שבהן מתחרה/אף-אחד מצוטט ואנחנו לא.
3. **מנסח patch** — מאמר/סקשן ממוקד שממלא בדיוק את הפער, ב-brand voice עברי.
4. **מפרסם ומפיץ** (publish + syndication, §3.5.3).
5. **מודד-מחדש** בקצב מוגדר → סוגר את הלולאה.

#### 3.5.2 מדד נוכחות-AI (Citation Score) + תנועת-AI (GA4)
- **Citation Score 0-100** per-אתר — מדד יחיד שמסכם נראות ב-AI (כמו 87/100 אצל enso).
- **Citation timeline** — גרף מגמה לאורך זמן.
- **Share-of-Voice** per-מנוע (ChatGPT/Perplexity/Claude/Gemini) — כמה מ-answer-set שייך לך מול מתחרים. **הרחבת כיסוי (🆕):** מנועים ללא API רשמי — **Google AI Mode · Copilot · Grok** — נמדדים דרך שכבת **BrightData scrape** (trigger→poll→parse citations), כך שהכיסוי מגיע ל-**7 מנועים** בלי להיות תלוי בזמינות API. (מקור-קצירה: `danishashko/geo-aeo-tracker`, MIT.)
- **AI Referral Traffic (GA4):** לא רק *נראות* אלא **תנועה אמיתית** — אילו עמודים מקבלים גולשים מ-ChatGPT/Perplexity/Gemini/Claude/Copilot (חיבור GA4 Data API + פילטר session-source, בהשראת שיטת ה-GA4). **הבידול:** רוב כלי ה-GEO מודדים ציטוטים; מעטים מודדים תנועה per-עמוד ממנועי AI. משלים את ה-Citation לתמונת ROI מלאה.

#### 3.5.3 מנוע קונצנזוס והפצה (Syndication) 🔗
מנועי AI ממליצים על מה שיש לו **corroboration** מחוץ לאתר (עיקרון מסקיל ה-GEO). לכן הסוכן לא רק מפרסם באתר — הוא **מפיץ** את התוכן/האזכורים לפלטפורמות צד-ג' (סושיאל, קהילות, דירקטוריות ישראליות). **מנצל את מנועי ההפצה של מוצרים 1+2** (וואטסאפ/טלגרם/מייל/סושיאל) — לא בונים מאפס.

#### 3.5.4 קצב הסוכן (Cadence) & Experiment Ledger
- **לולאת re-rank רציפה** — קצב מוגדר per-plan (למשל 6-24 שעות ב-plans גבוהים, שבועי ב-Starter). לא רק דוח חודשי.
- **Experiment Ledger** — הסוכן מריץ ניסויים מבוקרים (וריאציות כותרת/מבנה/schema/זווית) ו**לומד מה מנצח** לכל אתר; מתועד ל-`experiments` וניתן לצפייה. הליבה משתפרת per-לקוח לאורך זמן.

### 3.6 דשבורד ומעקב (Monitoring)
- **דירוגים בזמן אמת** (מ-GSC), TOP-10 tracking, מילות מפתח, תנועה, אימפרשנים/קליקים.
- **מדד נוכחות AI מאוחד** — **Citation Score (0-100)** + timeline + Share-of-Voice per-מנוע, לצד דירוגי גוגל (מבט מאוחד ייחודי).
- **לוח פערי ציטוט (Gap Board)** — משבצות לא-מאוישות פתוחות + סטטוס ה-patch שכל אחת קיבלה.
- **Experiment Ledger** — ניסויים שרצו, מה ניצח, ה-lift שנמדד (שקיפות + למידה).
- **נרטיב חודשי בעברית** — "החודש עלית ל-TOP 3 על 4 ביטויים ו-Citation Score עלה מ-61 ל-74; מומלץ לחזק תוכן בנושא X" (מנוע נרטיב משותף עם מוצר 2).
- **המלצות AI** — מה לכתוב הבא, איזה תוכן לרענן, איזה פער לסגור.

### 3.7 גישה מרובת-ערוצים (Multi-Channel Access) — נכנס + יוצא 🔀
לא רק דשבורד web — אפשר **להפעיל, לתשאל ולאשר** את הסוכן גם מהטלגרם ומהמייל, בלי להיכנס לאתר.

| ערוץ | גישה נכנסת (שליטה) | גישה יוצאת (דוחות/התראות) | סטטוס |
|---|---|---|---|
| **✈️ טלגרם** | בוט מלא — "מה הדירוגים שלי?", "אשר את המאמר", "כתוב על X", "עצור autopilot" | דוח חודשי + התראות (עלה ל-TOP 3 / מאמר פורסם / צוטט ב-ChatGPT) | ✅ שלב 5 |
| **📧 מייל** | פקודות ותגובות במייל — אישור מאמר בלחיצה, "reply to approve", בקשת נושא | דוח PDF חודשי + נרטיב בגוף ההודעה | ✅ שלב 5 |
| **📱 וואטסאפ** | תשתית מוכנה (adapter + טבלת opt-in) — הפעלה מלאה בהמשך | הודעת document מאושרת (utility template) | 🟡 תשתית בשלב 5, הפעלה בהמשך |

- **טלגרם ומייל** — ערוצי גישה מלאים (inbound + outbound) מ-day-one של שלב 5.
- **וואטסאפ** — נבנית **תשתית** (adapter, טבלת `channel_optin`, תבנית utility), ההפעלה המלאה בהמשך. ⚠️ דורש opt-in + תבנית מאושרת (§30A), לא ספאם.
- **מנצל את אותו מנוע הפצה של מוצרים 1+2** — אותם adapters, תשתית משותפת, cron edge function. בונים פעם אחת.
- **הסוכן כ-conversational agent:** אותו Claude שמריץ את הבוט עונה על שאלות בשפה טבעית בעברית דרך כל ערוץ.

### 3.8 מודולים לסגירת פערים מול השוק (parity + הרחבה) 🧩
מבוסס סקירת 18 כלים אמריקאים (§1.1). מוסיף 10 יכולות שקיימות אצל המתחרים וחסרו לנו — רובן מנצלות תשתית קיימת.

#### On-page & טכני
- **(1) אודיט + תיקוני SEO טכניים אוטומטיים** — הרחבת `geo-scan.ts` ל-Site Audit מלא (meta, redirects, broken links, alt-text, crawlability, schema — 200+ בדיקות) → תור תיקונים → **deploy per-CMS** (אותם adapters של §3.4) עם HITL או autopilot. סוגר את הפער מול OTTO/Alli/Frase.
- **(8) קישור פנימי אוטומטי** — הסוכן ממפה את מבנה האתר ומזריק קישורים פנימיים רלוונטיים per-מאמר (סמכות נושאית + זחילה).
- **(9) הגשה אוטומטית לאינדוקס** — דחיפת URLs חדשים ל-Google Indexing API / GSC לזירוז אינדוקס.

#### Off-page & Local
- **(2) בניית backlinks / digital-PR** — מנוע השגת אזכורים סמכותיים (outreach מנוסח ב-brand voice + דירקטוריות + הפצת קונצנזוס). ⚠️ scope גדול — שלב מתקדם.
- **(3) ניהול Google Business Profile** — עדכונים, פוסטים, ניהול רב-סניפי (local SEO). מנצל את מנוע ההפצה (adapter GBP כבר קיים ב-helix-ops).

#### הרחבות GEO
- **(4) AI-crawler analytics + pre-render** — מעקב תנועת בוטי-AI (GPTBot/ClaudeBot/PerplexityBot) + הגשת דף מרונדר לקרולרים (SSR snapshot). `geo-scan.ts` כבר בודק robots ל-AI crawlers — הרחבה טבעית.
- **(4b) AEO Audit — ציון מוכנות-לציטוט per-URL 🆕** — סורק **דף חי** ומחזיר ציון 0-100 של מוכנות לציטוט במנועי-AI לפי 11 בדיקות בשלוש קטגוריות: **גילוי** (llms.txt · llms-full.txt · גישת בוטי-AI ב-robots.txt · sitemap), **מבנה** (JSON-LD · FAQPage schema · meta · canonical), **תוכן** (תשובה-ישירה/BLUF · היררכיית כותרות). זה משלים את ה-Check Modes (§3.8.11, שבודק תוכן *לפני* פרסום) בכך שהוא בודק אתר *שכבר עלה*. מקור-קצירה: `danishashko/geo-aeo-tracker` (MIT), עברית-first.
- **(5) Prompt-volume data** — נתוני ביקוש: כמה אנשים באמת שואלים את ה-AI שאלה מסוימת (מזין את ה-prompt panel בתעדוף אמיתי, לא ניחוש). מקור: ספק דאטה (Semrush AI / Profound-like) או איסוף עצמי.
- **(6) סנטימנט בתשובות ה-AI** — האם ה-AI מדבר על המותג חיובי/ניטרלי/שלילי. **מנוע משותף עם HELIX Shield** (מוצר 5) — לא בונים פעמיים.

#### תוכן
- **(7) ניטור שחיקת-תוכן + רענון אוטונומי** (Content Watchdog) — מזהה מאמרים שדירוגם/תנועתם יורדים → מרענן אוטומטית (או מציע). סוגר פער מול Frase/Arvow.
- **(10) אינפוגרפיקות + באנרים דינמיים per-מאמר** — הרחבת מנוע התמונות (§3.3) מ-תמונת-hero בסיסית ל-אינפוגרפיקות/באנרים ממותגים. מנצל את `video-timeline.ts`/סטודיו של helix-ops.

#### מצבי-בדיקה ותאימות (Check Modes)
- **(11) מצבי-בדיקה per-plugin/תקן** — כמו WriterGPT: לפני פרסום, המאמר עובר סוללת בדיקות תאימות: **Rank Math** (50+), **Yoast**, **HCU (Helpful Content)**, **Semantic/NLP**, ומצב **Hybrid** משולב (76+ בדיקות). כל מאמר מקבל ציון + תיקונים אוטומטיים עד עמידה בסף. סוגר את הפער מול WriterGPT ומעלה אמינות מול לקוחות שמכירים Rank Math/Yoast.
- **סוכנים מומחים** (בהשראת WriterGPT): Content Writer · Editor · Technical SEO · Keyword Discovery · Internal Linking (§3.8.8) · Schema (§12.2) — כבר מכוסים ברובם; מצב ה-Check הוא ה-orchestrator שמריץ אותם עד ציון עובר.

> **⬜ White-label** — נדחה במפורש להמשך (כמו במוצרים האחרים).

### 3.9 GSC Intelligence — ניתוח אוטומטי של Search Console 🔎📊
מבוסס 7 הפרומפטים המוכחים ([nisai.dev/guides/gsc-ai-seo-prompts](https://nisai.dev/guides/gsc-ai-seo-prompts/)). ההבדל מהמדריך: המדריך = פרומפטים **ידניים** שאתה מריץ חודשי ואז עושה את העבודה בעצמך. HELIX Rank = **מריץ אוטומטית + מבצע את הפלט** (כותב FAQ, משכתב כותרת, מתקן קניבליזציה, מרענן דועך, מפרסם). 8 ניתוחים, כולם על **חיבור GSC API אמיתי** (לא chatbot שמנחש מספרים — יתרון אמינות מובנה):

| # | ניתוח | מה עושה | פעולה אוטומטית | סטטוס |
|---|---|---|---|---|
| 0 | **Health check** | מוודא שהדאטה אמיתית (לא הזיה) | חיבור API אמיתי | ✅ מובנה |
| 1 | **Striking Distance** ⚡ | מיקום 7-11.9 + ≥50 impressions → הרווח המהיר | משכתב Title/Meta/H1 + תוכן + קישור פנימי | 🆕 **מוסף** |
| 2 | **Executive Snapshot** | WoW/MoM, מה משותף למנצחים, כותרת אחת | נרטיב חודשי (§3.6) | ✅ |
| 3 | **מילון שאלות → FAQ** | שאלות (איך/מה/כמה/X מול Y) + פערי-תוכן | כותב FAQ 40-60 מילה + FAQ schema (GEO) | 🟡 מורחב |
| 4 | **מעקב שבועי** | wins/losses/new + 2 quick actions | דוח שבועי אוטומטי (§3.2) | ✅ |
| 5 | **קניבליזציה** 🔀 | כמה עמודים מתחרים על אותה שאילתה | מיזוג / דיפרנציאציה / 301 / קישור פנימי | 🆕 **מוסף** |
| 6 | **תוכן דועך** | 90d-over-90d, עמודים שירדו | רענון אוטומטי (Content Watchdog §3.8.7) | ✅ |
| 7 | **מפת דרכים 90 יום** | סינתזה לתכנית רבעונית מתועדפת | מזין את תור התוכן (§3.1) | ✅ |

- **2 הפערים החדשים (#1, #5) הם GSC-native** — לא צריך Semrush בשבילם, ROI גבוה, זולים לבנייה.
- **wow ל-onboarding:** רגע שהלקוח מחבר GSC → הבוט מחזיר מיד דוח Striking Distance ("5 מילים במקום 8 שאפשר לקפוץ ל-3 השבוע"). ה-hook שממיר.

## 6. ארכיטקטורה
- **Frontend:** React + shadcn (על צורת PLUG, RTL מוכן) / Next.js לאתר.
- **Backend:** Supabase (Auth, DB, Storage, RLS per-site).
- **Research:** edge function — GSC API + **Semrush/Ahrefs API** (volume/difficulty/מתחרים) + FireCrawl (`geo-scan.ts` קיים) → `keywords`. cache per-ביטוי כדי לחסוך קריאות API יקרות.
- **Content:** Claude + Hebrew Content by Helix (+ הגהת baldiga חובה) → `content_pieces`; תמונות דרך image-gen.
- **Publishing:** adapter per-CMS (edge function) — WordPress REST / Wix / Webflow / webhook.
- **GEO:** `geo-scan.ts` ל-citation tracking + מחולל `llms.txt` + **AI-crawler analytics/pre-render** (§3.8.4).
- **Site Audit + fix (§3.8.1):** edge function מרחיב את `geo-scan.ts` ל-200+ בדיקות → `site_issues` → תור תיקונים → deploy per-CMS.
- **סנטימנט AI (§3.8.6):** מנוע משותף עם HELIX Shield (מוצר 5).
- **Cron:** edge functions (`--no-verify-jwt`) — drip פרסום, סריקת דירוגים שבועית, **Content Watchdog** (שחיקה→רענון), auto-index, דוח חודשי.
- **Delivery + גישה נכנסת:** מנוע ההפצה המשותף (מייל/טלגרם, תשתית וואטסאפ) ממוצרים 1+2. טלגרם דרך Bot API (webhook → edge function → Claude); מייל דרך inbound-parse (Resend/Postmark) לאישורים ופקודות; וואטסאפ Cloud API — adapter מוכן, הפעלה בהמשך.

## 12. לוגיקת הסוכן (Agent Logic) — בסיס ל-system prompt
מזוקק משני סקילים: **`seo-content-strategist`** (קיים) + **`geo-content-strategist`** (נבנה עכשיו, מעוגן ב-`geo-scan.ts`). זו הליבה שהופכת ל-system prompt של ה-agent בפרודקשן.

### 12.1 מנוע המחקר (Research) — SEO
- **Seed → הרחבה → סיווג → תעדוף → מיפוי.** seed 5-10 ביטויים מהמוצר/לקוח → הרחבה (מתחרים, PAA, related, שאלות) → סיווג כוונה → תעדוף → הקצאה למאמר.
- **Keyword Opportunity Score** = `(Volume × Intent × ClickRate) / Difficulty` (Info=1, Commercial=3, Transactional=5). ממיין את תור התוכן.
- **סף difficulty לפי גיל/סמכות האתר:** אתר חדש → מטרות 0-20 (1-3 חודשים); מבוסס → 21-40. לא לרדוף head terms בלתי-נגישים.
- **פילוח כוונה** (Informational/Commercial/Transactional) קובע סוג תוכן: מדריך / השוואה / דף המרה.
- **ניתוח פער מתחרים:** ביטויים שהם מדרגים עליהם ואנחנו לא, volume>100 → תוכן חדש/משופר.

#### 12.1.1 GSC Intelligence — לוגיקת הניתוח (בסיס ל-system prompt, §3.9)
מזוקק מ-7 הפרומפטים המוכחים. כל ניתוח = query→page pairs מ-GSC (90 יום), ואז פעולה:
- **Striking Distance:** `position ∈ [7.0, 11.9] AND impressions ≥ 50` → בחר 3-5 עם ה-upside הגבוה → לכל אחד: Title/Meta/H1 מדויקים חדשים + מה חסר מול top-3 + קישור פנימי + anchor. ממויין easiest-win-first.
- **CTR נמוך = בעיית כותרת:** impressions גבוה + CTR מתחת לממוצע האתר → שכתוב Title/Meta (התיקון הזול ביותר).
- **קניבליזציה:** ≥2 URLs שלנו על אותה שאילתה → זהה primary (intent+ביצועים) → מיזוג/301/דיפרנציאציה/קישור פנימי. תעדוף לפי impressions מבוזבזים שאף עמוד לא ב-top-3.
- **שאלות → FAQ:** סנן שאלות (איך/מה/כמה/האם/X מול Y) → תשובת 40-60 מילה לכל אחת → היכן לשכן (עמוד קיים/חדש) → **flag פערי-תוכן** (שאלה עם impressions בלי עמוד).
- **תוכן דועך:** 90d-over-90d, עמודים שאיבדו קליקים/מיקום → סיבה משוערת → תכנית רענון.
- **תמיד:** לאמת מספרים גולמיים מול GSC לפני פעולה (anti-hallucination), ולתת ל-AI קונטקסט עסקי ("חנות תכשיטים, ישראל בלבד") לתעדוף מדויק.

### 12.2 מנוע התוכן (Content) — SEO + GEO ביחד
- **מבנה answer-first:** כל H2 = שאלה אמיתית; 40-60 מילים ראשונות = תשובה מלאה ועצמאית (מנצח גם featured snippet וגם ציטוט AI).
- **Topic clusters:** pillar page (3k+ מילים) + מאמרי cluster שמקשרים אליו — סמכות נושאית.
- **claim density:** טענה אחת למשפט, עובדתית, עם ייחוס — כדי שמנוע AI ירים את המשפט ויַחֵס אותו למותג.
- **schema אוטומטי:** Article + FAQPage + HowTo per-מאמר; Organization/LocalBusiness site-wide עם `sameAs` (עוגן ישות).
- **GEO built-in:** `llms.txt` אוטומטי, robots.txt שלא חוסם GPTBot/ClaudeBot/PerplexityBot/Google-Extended, ותוכן server-rendered שקריא לקרולר.
- **דו-לשוני:** עברית → ניתוב חובה דרך Hebrew Content + הגהת `baldiga` (הגנה מפני קנס "AI spam" ומפני "ריח AI"); אנגלית → Claude פרוזה סטנדרטית. שפת ה-prompt panel וה-schema נגזרת מ-`content_lang` של האתר.

### 12.3 מנוע SERP/GEO features
- **מבנה לזכייה ב-position 0:** פסקה (הגדרה) / רשימה (שלבים) / טבלה (השוואה) לפי סוג ה-SERP.
- **PAA mining:** לענות על 5-10 שאלות PAA כ-H2 ישירים בכל מאמר.
- **GEO Layer B (דטרמיניסטי, תדיר):** אימות crawlability/schema/llms.txt דרך `geo-scan.ts`.
- **GEO Layer A (הסתברותי, מתוזמן):** prompt panel של 20-50 שאילתות בעברית × 4 מנועים → **Share of Answer** ומגמה, לא מספר מוחלט.
- **"ציטוטים ברמת ויקיפדיה":** הזרקת מקורות סמכותיים מיוחסים לתוכן → מגדילה את הסיכוי שה-LLM יצטט אותנו כמקור.

### 12.3.1 לולאת Citation-Gap → Patch (הליבה האגנטית)
`scan(prompt panel בשפת האתר) → detect gaps (משבצות לא-מאוישות) → draft patch (brand voice) → publish + syndicate → re-rank`. רץ בקצב per-plan. כל gap נשמר ב-`citation_gaps` עם סטטוס; כל תיקון מקושר ל-`content_piece`. זו ההשראה מ-enso/Hermes, בעברית-first.

### 12.3.2 Experiment Ledger (למידה)
הסוכן מנסח השערה (וריאציית כותרת/מבנה/schema/זווית), מריץ, מודד lift, ו**שומר את המנצח** ל-`experiments`. הליבה מתכווננת per-אתר לאורך זמן — לא prompt סטטי.

### 12.4 לולאת מדידה (Measure → Iterate)
- SEO: דירוגים (GSC), CTR, TOP-10, תנועה אורגנית → רענון תוכן שנשחק (רבעוני).
- GEO: **Citation Score** + Share of Answer per-מנוע + מגמה → אם ingestible אך לא מצוטט = לפתוח gap ולהריץ patch + syndication (§12.3.1).

---
**סיכום:** לא "עוד בוט SEO". **בוט דו-לשוני (עברית-first + אנגלית) עם SEO + GEO שווי-ערך** (מדרג גם בגוגל וגם במנועי AI), עם **מנוע Citation-Gap→Patch אגנטי** ברמת enso/Hermes אך עברית-first, **שמפרסם לכל פלטפורמה (any-CMS)**, עם **תוכן אנושי אמיתי (baldiga)**, ו**גישה מרובת-ערוצים** (web/טלגרם/מייל, תשתית וואטסאפ) — צירוף בידולים שאף מתחרה (seo-agent.io / enso.bot) לא נותן. יושב על תשתית HELIX הקיימת, נמכר בתשלום ישיר תחת מותג HELIX, ומזין את כל האקו-סיסטם.

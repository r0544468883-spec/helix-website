# HELIX Bots — שיח ותפעול (Conversation & Operations) 🤖💬

> שכבת שיח **משותפת לכל בוטי HELIX** (Rank, Shield, וכל האקו-סיסטם) על **טלגרם · וואטסאפ · מייל**.
> שני צירים: **הקמה/תשתית** (חלק א') + **כתיבה וסדר-עבודה של השיח** (חלק ב').
> עברית-first (baldiga חובה על כל טקסט עברי) · תשתית אחת, נבנית פעם אחת · תאריך: 2026-07-18.

---

## 0. עיקרון-על
כל הבוטים חולקים **מנוע שיח אחד**: webhook → Claude conversational agent → tools. ההבדל בין Rank ל-Shield הוא ה-*tools* וה-*system prompt*, לא ערוץ התקשורת. אותו קוד ערוצים, אותה ספריית microcopy, אותם templates — משתנה רק התוכן. זה חלק מ**החיבור לאקו-סיסטם HELIX** (ראה קבצי המוצר).

# חלק א' — הקמה ותשתית (Setup)

## 1. טלגרם (Telegram Bot API)
הכי פשוט להקים — נקודת הכניסה המומלצת ל-day-one.
- **יצירה:** `@BotFather` → `/newbot` → מקבלים **bot token**.
- **קבלת הודעות:** webhook — `setWebhook` ל-edge function (`POST /telegram/webhook`). מחזירים 200 מיד, מעבדים אסינכרוני.
- **פקודות:** `setMyCommands` לרישום תפריט הפקודות (§5). תומך גם שפה טבעית (לא רק slash).
- **כפתורים:** `inline_keyboard` לאישור/דחייה בלחיצה (approve/reject).
- **זהות:** ממפים `chat_id` ↔ `subject_id`/`site_id` דרך `channel_bindings`. אימות ראשוני בקוד חד-פעמי.
- **בלי חלון 24 שעות** — טלגרם מאפשר הודעות יזומות חופשי (יתרון על וואטסאפ). כאן שולחים התראות/דוחות בלי מגבלה.

## 2. וואטסאפ (WhatsApp Cloud API)
העוצמתי ביותר, אבל **הכי מוסדר**. תשתית נבנית עכשיו, הפעלה מלאה בהמשך.
- **בסיס:** `POST https://graph.facebook.com/v21.0/{phone-number-id}/messages`, `Authorization: Bearer {token}`, `"messaging_product":"whatsapp"`.
- **מספר טלפון:** WABA + phone-number-id ייעודי. פורמט נמענים **E.164** (`+972…`, בלי רווחים/מקפים).
- **webhook:** אימות GET (`hub.challenge`) + POST להודעות/סטטוסים. להחזיר 200 מיד; לטפל ב-idempotency (אותו event עלול להגיע פעמיים). לשמור `wamid`.
- **חלון 24 שעות (קריטי):**
  - לקוח כתב לבוט → נפתח **חלון שירות 24 ש'** → בתוכו כל סוג הודעה **חינם**.
  - מחוץ לחלון → **רק template מאושר** (בתשלום per-message). זה המנגנון לדוחות/התראות יזומים.
- **opt-in חובה** לפני הודעה יזומה — נשמר ב-`channel_optin` (מקור הסכמה, timestamp). תואם **§30A** (חוק הספאם הישראלי): הלקוח בחר לקבל.
- **rate limit:** 80 הודעות/שנייה → תור + backoff. שגיאות מפתח: `131047` (re-engagement → שלח template), `131050` (opt-out → כבד, שירות/utility בלבד).

### 2.1 קטלוג templates (קטגוריית Utility)
דוחות/התראות יזומים = **utility templates** מאושרים מראש (לא marketing — utility עובר אישור מהר ולא נחשב שיווקי). מגישים ל-`POST /{WABA_ID}/message_templates`, ממתינים ל-APPROVED.

| template | קטגוריה | שימוש | משתנים |
|---|---|---|---|
| `helix_report_ready` | utility | "הדוח {{1}} שלך מ-HELIX מוכן" + כפתור URL | bot, period, url |
| `helix_approval_request` | utility | בקשת אישור מאמר/הסרה + כפתורי QUICK_REPLY | subject, item, type |
| `helix_crisis_alert` | utility | Shield — אזכור שלילי חדש בעל סמכות | name, source, url |
| `helix_published` | utility | "המאמר '{{1}}' פורסם ומדורג לניטור" | title |
- כפתורים: `QUICK_REPLY` (מחזיר payload ל-webhook → אישור/דחייה) או `URL` (לינק לדשבורד).
- ⚠️ template = השלד המאושר; ה**שיח החופשי** קורה רק בתוך חלון 24 ש' אחרי שהלקוח הגיב.

## 3. מייל (Inbound + Outbound)
- **יוצא:** Resend/Postmark — דוח PDF + נרטיב בגוף (מנוע ההפצה המשותף).
- **נכנס:** inbound-parse (Resend/Postmark webhook) → edge function. **reply-to-approve:** תשובה "אשר"/"דחה" למייל בקשת-אישור סוגרת את הלולאה בלי להיכנס לאתר.
- מיפוי כתובת → `channel_bindings`.

## 4. ארכיטקטורת השיח (משותפת)
```
ערוץ (Telegram/WA/Email webhook)
  → edge function (normalize → זהות מ-channel_bindings)
  → Claude conversational agent (system prompt per-bot + היסטוריה)
  → tools (status / write / approve / removal / report…)
  → תגובה בערוץ (microcopy מהספרייה המשותפת)
```
- **idempotency + 200 מיד** בכל הערוצים.
- **human-in-the-loop:** פעולות רגישות (פרסום, הסרה, שליחה יזומה) עוברות אישור אנושי דרך הבוט.

# חלק ב' — כתיבה וסדר-עבודה (Conversation Design)

## 5. אוצר פקודות (Command Vocabulary)
עברית-first, פועל+שם, עקבי בכל הערוצים. slash בטלגרם / שפה טבעית בכולם.

**משותף (כל הבוטים):**
| פקודה | טבעי | מה עושה |
|---|---|---|
| `/status` | "מה המצב?" | סיכום מצב נוכחי + מדד מפתח |
| `/report` | "שלח דוח" | דוח אחרון (PDF/סיכום) |
| `/approve` | "אשר" | מאשר את הפריט הממתין |
| `/reject` | "דחה" | דוחה + סיבה |
| `/pause` · `/resume` | "עצור" · "המשך" | autopilot |
| `/help` | "עזרה" | רשימת פקודות |

**Rank (SEO+GEO):** `/write <נושא>` ("כתוב על…"), `/gaps` (פערי ציטוט), `/score` (Citation Score), `/rankings`.
**Shield (מוניטין):** `/threats` (איומים פתוחים), `/removals` (בקשות הסרה), `/reputation` (Reputation Score), `/alerts`.
**OPS/Engage (Marketing Ops):** `/funnels` (Comment-to-DM funnels), `/leads` (לידים שנלכדו), `/engage` (הצעות תגובה), `/exposure` (מד-חשיפה + kill-switch), `/publish` (פרסום/תזמון).

## 6. עקרונות השיח (מ-ux-writer, מותאם עברית)
1. **תפקיד אחד להודעה** — ליידע / להנחות / לאשר / להזהיר. לא לערבב.
2. **קצר ואנושי** — משפט-שניים. פועל+שם בכפתורים ("אשר מאמר", לא "אישור").
3. **נוסחת שגיאה:** מה קרה → למה (אם עוזר) → מה לעשות. בלי להאשים את המשתמש.
4. **עקביות** — אותו פועל לאותה פעולה בכל הבוטים והערוצים.
5. **הקשר בכל הודעה יזומה** — הבוט פונה ראשון, אז חייב לומר על מה מדובר ("על השם שלך", "המאמר X").
6. **baldiga חובה** — כל טקסט עברי עובר הגהת כתיב לפני שנכנס לספרייה.

## 7. סדר-עבודה של השיח (Work-Order / Turn Structure)
מבנה תור אחיד לכל אינטראקציה יזומה:
```
1. TRIGGER   — אירוע (מאמר מוכן / אזכור שלילי / דוח חודשי)
2. NOTIFY    — הודעה קצרה + הקשר + הפריט
3. ASK       — שאלה/פעולה אחת ברורה + כפתורים (אשר/דחה/הצג)
4. CONFIRM   — אישור הפעולה + הצעד הבא
5. LOG       — עדכון סטטוס בטבלה (approved/published/removed…)
```
בשיח נכנס (משתמש יוזם): NLU → tool → תשובה קצרה + פעולת-המשך אופציונלית.

## 8. ספריית Microcopy (עברית)

### 8.1 Rank — אישור מאמר
> **TRIGGER:** מאמר מוכן לפרסום.
- **NOTIFY+ASK:** "מאמר חדש מוכן: *«5 טעויות SEO שכל עסק עושה»* (1,940 מילים, עברית). לפרסם?"
  כפתורים: **[אשר ופרסם] [ערוך] [דחה]**
- **CONFIRM (אישר):** "פורסם ✅ המאמר עלה ל-blog ונכנס לניטור דירוגים. אעדכן כשייכנס ל-TOP 10."
- **CONFIRM (דחה):** "בוטל. רוצה שאכתוב גרסה אחרת, או נדלג על הנושא?"

### 8.2 Shield — אישור בקשת הסרה
> **TRIGGER:** זוהתה תוצאה שלילית עם ערוץ-הסרה זמין.
- **NOTIFY+ASK:** "מצאתי תוצאה שלילית על השם שלך ב-*example.co.il* (עמוד 1, מקום 4). ערוץ אפשרי: פנייה מנומקת לבעל האתר. לשלוח?"
  כפתורים: **[אשר שליחה] [הצג טיוטה] [דחיקה במקום]**
- **CONFIRM:** "הבקשה נשלחה 📨 סטטוס: *ממתין לתגובה*. אעדכן אם יוסר. במקביל התחלתי לבנות תוכן חיובי לדחיקה."
- **handoff משפטי:** "המקרה הזה נראה כמו לשון הרע שמצריך **עורך דין** — זה מעבר לשירות שלי. רוצה שאסמן אותו להפניה?"

### 8.3 Shield — התראת משבר (יזום)
> **TRIGGER:** אזכור שלילי חדש בעל סמכות גבוהה.
- **ALERT:** "🔴 התראה: אזכור שלילי חדש על *«שם הלקוח»* פורסם היום ב-*ynet* ומדורג עולה. רוצה שאתחיל טיפול?"
  כפתורים: **[הצג] [התחל טיפול] [התעלם]**

### 8.4 מצבי שגיאה (נוסחת what→why→fix)
| מצב | הודעה |
|---|---|
| ערוץ לא מחובר | "עוד לא חיברת את חשבון ה-GSC — בלעדיו אין דירוגים. לחבר עכשיו?" |
| פקודה לא ברורה | "לא הבנתי. אפשר: 'מה המצב', 'שלח דוח', או 'כתוב על…'. או שלח /help." |
| פעולה נכשלה | "הפרסום נכשל — האתר לא הגיב. ניסיתי שוב ולא עבר. אפשר לבדוק את החיבור ל-WordPress?" |
| מחוץ לחלון (WA) | *(אוטומטי)* נשלח template `helix_report_ready` במקום הודעה חופשית. |

### 8.5 הצלחה / ריק
- **הצלחה:** "בוצע ✅ [+ הצעד הבא במשפט אחד]".
- **ריק (אין מה להציג):** "הכל שקט — אין איומים/פערים פתוחים כרגע. אעדכן ברגע שיהיה משהו."

## 9. Guardrails בשיח
- **אישור לפני פעולה בלתי-הפיכה** — פרסום, שליחת בקשת הסרה, הודעה יזומה ללקוח-קצה.
- **גבולות אתיים (Shield)** — סירוב מנוסח: "לא אוכל לסייע להסתיר תוכן עיתונאי לגיטימי. אני יכול להתמקד ב-PII, ביקורות מזויפות, ותוכן שמפר מדיניות."
- **opt-out מכובד** — "הפסק" בכל ערוץ → עצירת הודעות יזומות מיידית (WA `131050`, §30A).
- **שקיפות** — הבוט תמיד אומר מה עשה ומה הסטטוס; לא פועל בשקט.

## 10. סדר בנייה (Roadmap לשכבת השיח)
| שלב | מה בונים |
|---|---|
| **1** | טלגרם — webhook + פקודות + זהות + microcopy בסיסי (הכי מהיר) |
| **2** | מייל — inbound-parse + reply-to-approve + דוחות יוצאים |
| **3** | וואטסאפ — Cloud API, webhook, opt-in, 4 utility templates, חלון 24 ש' |
| **4** | מנוע NLU מלא (Claude conversational) חוצה-ערוצים + ספריית microcopy מלאה (baldiga) |
| **5** | Guardrails + human-in-the-loop + opt-out + התראות משבר |

---
**סיכום:** שכבת שיח **אחת** לכל בוטי HELIX. **הקמה:** טלגרם (הכי פשוט, בלי חלון) → מייל → וואטסאפ (חזק, מוסדר, utility templates + חלון 24 ש' + opt-in §30A). **כתיבה:** עברית-first (baldiga), פועל+שם, תפקיד-אחד-להודעה, מבנה תור אחיד (Trigger→Notify→Ask→Confirm→Log), עם guardrails ואישור אנושי. משתנה בין הבוטים רק ה-tools וה-system prompt — לא הערוצים ולא ה-microcopy engine.
